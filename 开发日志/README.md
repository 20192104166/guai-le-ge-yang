# 开发日志

> 每日工作记录。是项目"心跳"，确保每次会话都有上下文、有交接、有追踪。

---

## 1. 命名规则

- 文件名：`YYYY-MM-DD.md`（如 `2026-05-15.md`）
- 一天一份
- **不允许**：跨天合并、跳过日期（哪怕当天无开发也要写一份"今日空闲"占位）

---

## 2. 维护时机

| 时机 | 动作 |
|---|---|
| 会话开始 | 阅读最近一份日志，新建当日日志（如未建） |
| 完成任务 | 实时勾选 ✅，记录产出物 |
| 重要决策 | 实时写入「💡 决策记录」 |
| 遇到阻塞 | 实时写入「⚠️ 风险与阻塞」 |
| 会话结束 | 补齐当日总结、TODO 转交明日 |

---

## 3. 日志模板

每份日志使用以下结构（也可复制 [`模板.md`](模板.md)）：

```markdown
# YYYY-MM-DD 开发日志

**📅 当前阶段**：Phase X - <阶段名>
**👤 协作者**：用户 / Claude

---

## ✅ 今日完成
- [x] T0.x <任务描述> → <产出物链接>

## 🚧 进行中
- [ ] T0.x <任务描述>（进度 XX%）

## 📌 明日待办（TODO）
- [ ] <任务>

## ❓ 待确认问题
- <问题> → 等待用户确认

## 💡 决策记录
- <决策描述>（原因：xxx）

## ⚠️ 风险与阻塞
- <风险描述>（应对方案：xxx）

## 📝 备注
- 任何零散的想法、参考链接、灵感
```
