## 2. 弹沙嘎天梯榜：ELO 段位

### 2.1 6 级段位（蒙古主题）

| 等级 | 段位名 | ELO 分段 | 赛季奖励 |
|---|---|---|---|
| 1 | 牧羊童 | 0 ~ 1000 | 头像框 |
| 2 | 套马者 | 1001 ~ 1300 | 头像框 + 1 张普卡 |
| 3 | 草原骑兵 | 1301 ~ 1700 | 段位徽章 + 2 张普卡 |
| 4 | 巴特尔（勇士）| 1701 ~ 2100 | 段位徽章 + 1 张稀有卡 |
| 5 | 苍狼之主 | 2101 ~ 2500 | 限定皮肤 + 1 张稀有卡 |
| 6 | 长生天眷顾者 | 2501+（赛季前 100 名）| 限定特效 + 限定皮肤 + 大师卡 |

### 2.2 ELO 算法

标准 ELO 公式：

```
预期胜率 E_A = 1 / (1 + 10^((R_B - R_A) / 400))
新分数 R_A' = R_A + K × (S_A - E_A)
  其中 S_A: 实际结果（胜=1, 负=0, 平=0.5）
       K:   常数 32
```

- 起始 ELO = 1000
- K 值 = 32（标准值，便于段位变动有感）
- 上限段位（长生天眷顾者）K 值降到 16，使顶端波动小

### 2.3 赛季机制

- **赛季周期**：每月 1 号 0 点重置
- **重置规则**：
  - ELO 下沉 = max(1000, 当前 ELO × 0.9)
  - 段位至少保留前一赛季 -1 段位
- **赛季奖励发放**：
  - 赛季结束当日发放上赛季最高段位奖励
  - 邮件 / 弹窗通知

### 2.4 防作弊

- 仅 **匹配制对战** 计 ELO（系统随机匹配）
- 好友赛、邀请赛 **不计** ELO
- 同 IP 短时多次匹配 → 风控阻断
- AI 对手 **不计** ELO

### 2.5 段位徽章设计原则

参考 [`../03-设计规范/美术风格指南.md`](../03-设计规范/美术风格指南.md)：
- 牧羊童：小孩 + 绵羊 Q 版剪影
- 套马者：套马杆图标
- 草原骑兵：骑马轮廓
- 巴特尔：盾牌 + 弓
- 苍狼之主：狼头 + 月亮
- 长生天眷顾者：金色苍鹰 + 五色风马旗

## 3. 抛沙嘎手气榜：周榜单局最高分

### 3.1 排序

- 仅按"当周单局最高分"排序
- 每周一 0 点清零
- 单玩家同一周内多次刷新只保留最高

### 3.2 触发"上榜"事件
- 单局结束时与本周最高分对比
- 刷新最高 → 上榜 + 弹特效
- 触发"撂四样" / "四骆驼" → 上榜时携带特殊标签
- 单局 ≥ 1000 分 → 推荐进入"高光时刻"

### 3.3 双轨展示
- 当周榜（主榜）
- 历史最高分榜（次要展示，玩家个人页面）

### 3.4 周清零逻辑
- cron `cron_weekly_reset` 每周一 0 点执行
- 备份上周 top 100 到 `leaderboards` 集合（type = "toss_week"）
- 重置所有 user 的 `tossWeekBest` = 0

## 4. 抓沙嘎挑战榜：双榜

### 4.1 永久榜：最高通关数

- 排序：`catchMaxLevel desc`
- 同关数比较通关用时（越短越前）
- 永久保存

### 4.2 周榜：最快通关 5 关用时

- 排序：`catchWeekBestTime asc`（只统计完整通过 5 关的成绩）
- 仅当完成 5 关时记录用时
- 周清零

### 4.3 关卡组扩展
后续每个赛季可新增"赛季限定关卡组"（用蒙古节日主题，如那达慕篇、白月篇）。

## 5. 沙嘎大师榜（综合）

### 5.1 大师分计算公式

```
masterScore = (flickElo - 1000) × 0.4
            + tossBestScore × 0.3
            + catchScoreUnit × 0.3

其中：
  catchScoreUnit = catchMaxLevel × 100 + max(0, 600000 - catchWeekBestTime) / 100
  （通过关卡数 × 100，加上"通关用时越短越高"的奖励）
```

### 5.2 加权调整原则
- 三玩法均衡（0.4 / 0.3 / 0.3）
- 弹沙嘎权重稍高（PvP 含金量）
- 任何玩家若只玩一个玩法不能登顶（鼓励"全能"）

### 5.3 季度榜单展示
- 主榜单永久
- 季度榜单单独展示（每季度末归档）
- 前 100 名公开展示，101+ 私密

## 6. 好友榜

### 6.1 数据来源
- 抖音小游戏 API：`tt.getFriendCloudStorage` / `tt.getUserCloudStorage`
- 服务端 `friendships` 缓存

### 6.2 展示
- 默认显示玩家好友的"大师分"排序
- Tab 切换：大师分 / 弹 ELO / 抛最高 / 抓最高
- 高亮自己的名次
- 一键发起"挑战好友"

### 6.3 隐私
- 玩家可在设置中"隐藏自己"，则不在好友榜显示
- 默认开启显示

## 7. 邀请好友 PK 流程

### 7.1 邀请发起

```
玩家 A → 主菜单 → 选择玩法 → 点"邀请好友"
  → 弹出好友列表（来自抖音 SDK）
  → 选定好友 → 调云函数 `pvp_invite_friend` 生成分享卡片
  → 触发抖音分享 API（卡片 + 跳转链接含 challengeId）
```

### 7.2 邀请接收

```
玩家 B 在抖音收到分享卡片
  → 点击 → 跳转小游戏 → 携带 challengeId 进入
  → 自动展示"X 邀请你 PK [玩法]"
  → 同意 → 进入 PvP 异步流程（见服务端架构 §3.2）
```

### 7.3 邀请奖励
- 双方完成挑战均获得"好友切磋"积分（不计 ELO）
- 累积 N 次解锁限定头像框

### 7.4 反作弊
- 同好友 24 小时内挑战次数限制
- 邀请链接 7 天有效
