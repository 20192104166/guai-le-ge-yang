# 开发路线图

> 版本：v0.2（已根据 Unity 2D + 后端需求重排）
> 原则：**稳定 · 安全 · 有效**。每个 Phase 必须达成"可验证、可演示、可回退"才进入下一 Phase。
> 当前阶段：**Phase 0 - 项目筹备**

---

## 总体阶段规划（v0.2）

```
Phase 0 → Phase 1 → Phase 1.5 → Phase 2 → Phase 3 → Phase 4 → Phase 5 → Phase 6 → Phase 7
  筹备     物理预研   后端预研    首玩法MVP  抓沙嘎   弹沙嘎   美术 & UI  商业化 & 社交  测试上线
```

> v0.2 关键变化：
> - 增加 **Phase 1.5 后端预研**（PvP / 排行榜 / 数据持久化需要服务端）
> - Phase 3/4 顺序：抓沙嘎在前（单人易实现），弹沙嘎在后（涉及 AI/PvP 最复杂）

## Phase 1 - 物理 POC（前端预研）

**入场条件**：Phase 0 完成
**预计周期**：1 ~ 2 周
**目标**：在 Unity 2D 环境验证「不规则羊拐 2D 物理 + 加权随机」可行性

### 关键任务
1. **Unity 2D 项目骨架**：建立最简空场景
2. **羊拐 2D 美术占位**：4 面各画一张占位贴图
3. **物理模型**：用 `PolygonCollider2D` 简化羊拐外形
4. **物理 + 加权随机 混合**：
   - 物理模拟翻滚动画
   - 静止时按设定概率随机最终面
   - 平滑过渡动画
5. **统计验证**：自动抛 1000 次，统计落地分布
6. **抖音运行环境性能验证**：在 Unity For Mini Games 插件下打包测试

### 交付物
- 可演示 Demo（4 枚羊拐抛掷 + 统计面板）
- 落地分布报告
- 包体 / 帧率报告

### 验收标准
- 落地分布与目标比例误差 ≤ 5%
- 抖音运行帧率 ≥ 50 FPS
- 主包体积 ≤ 5MB（含基础 UI）
- "翻面过渡"动画体感自然，无明显穿帮

### 风险预案
- 若 Unity WebGL 抖音端性能差 → 退路：Cocos Creator
- 若"加权随机"被察觉 → 退路：全物理（概率不可控但真实）

## Phase 1.5 - 后端预研

**入场条件**：Phase 1 完成 + E1/E2 已确认
**预计周期**：1 周
**目标**：搭建抖音云开发（或选定方案）基础后端能力

### 关键任务
1. 注册抖音云开发，创建项目
2. 搭建云数据库 schema：
   - 用户表（uid、昵称、头像）
   - 羊拐 / 大师分 / 段位 资产表
   - 排行榜表（三玩法分榜）
   - 图鉴解锁表
   - PvP 录像表
3. 编写云函数原型：登录、查询排行榜、上传录像
4. Unity 客户端集成 SDK（抖音登录、云数据库读写）

### 交付物
- 可工作的"登录 → 上传一条数据 → 拉取榜单"完整链路 demo

### 验收标准
- 抖音账号一键登录通过
- 数据增删查改正常
- 排行榜可见（用 mock 数据填充）

## Phase 2 - 首玩法 MVP（抛沙嘎）

**入场条件**：Phase 1 + Phase 1.5 完成
**预计周期**：3 ~ 4 周
**目标**：以抛沙嘎打通"单机 + 异步 PvP + 排行榜上传"完整闭环

### 关键任务
1. 抛掷手势（拖拽蓄力）交互实现
2. 落地形态自动识别
3. **完整计分引擎**（按 [`玩法-抛沙嘎.md`](../06-游戏设计/玩法-抛沙嘎.md) §3 计分表）
4. 加成机制实现
5. 简版蒙古风 UI（开始、得分、结算）
6. 单机 AI 对手（简单难度）
7. 单局录像系统（操作序列）
8. 异步 PvP：录像上传 / 下载 / 回放
9. 排行榜接入（手气榜）
10. 激励视频广告接入（复活、双倍）

### 交付物
- 完整可玩的抛沙嘎单玩法版本（含 PvP 异步）
- 玩家可看到自己在手气榜的排名

### 验收标准
- 完整闭环可玩 10 局无 BUG
- 计分逻辑与设计文档 100% 一致
- 异步 PvP 录像可正确回放
- 蒙古风占位美术统一

---

## Phase 3 - 抓沙嘎

**入场条件**：Phase 2 验收通过
**预计周期**：2 ~ 3 周
**目标**：扩展抓沙嘎玩法（关卡模式 + 双榜）

### 关键任务
1. 沙包抛接物理 + 计时机制
2. 多指 / 拖拽快速摆位交互
3. 形态校验（是否摆到目标）
4. 关卡数据驱动（5 关一组 + 无尽模式）
5. 双榜接入（永久 + 周榜）
6. 时光回溯广告位

### 验收标准
- 关卡梯度合理（用户测试 5 人均能通过第 1 关）
- 操作响应延迟 ≤ 100ms
- 与抛沙嘎共享物理 / 素材资源

## Phase 4 - 弹沙嘎

**入场条件**：Phase 3 验收通过
**预计周期**：3 ~ 4 周
**目标**：实现最复杂的弹沙嘎（含 ELO 段位）

### 关键任务
1. 弹射手势 + 轨迹预测
2. 弹射物理 + 碰撞判定
3. 同形态识别与赢取规则
4. AI 对手（4 难度档）
5. 局制比赛（输完即败）
6. 异步 PvP 集成
7. ELO 段位系统（赛季制）
8. 段位徽章 + 蒙古主题段位名
