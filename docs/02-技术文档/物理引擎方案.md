## 0. 方案总览

```
玩家操作
   ↓
力度 / 方向输入
   ↓
[Unity Box2D 真实物理模拟] ← 翻滚 / 碰撞 / 摩擦
   ↓
检测"几乎静止"（速度阈值 + 持续时间）
   ↓
[加权随机决定最终面 F]    ← 概率分布表
   ↓
[平滑过渡动画]            ← 物理被禁用 → 朝向插值到 F
   ↓
触发"形态判定回调" → 计分 / 命中判定 / 抓沙嘎校验
```

**核心思想**：
- 物理引擎负责**视觉表现**（翻滚、碰撞、自然衰减）
- 加权随机负责**结果公平性**（每次抛掷不能因某玩家手机帧率不稳就改变概率）
- 过渡动画掩盖"伪物理"的痕迹

## 1. Unity Box2D 配置

### 1.1 项目配置（Project Settings → Physics 2D）

| 参数 | 推荐值 | 说明 |
|---|---|---|
| Gravity | (0, -30) | 略大于真实 9.8，使翻滚更快、节奏更紧凑 |
| Default Material | 单独定义 | 见 1.2 |
| Velocity Iterations | 8 | Box2D 默认 |
| Position Iterations | 3 | Box2D 默认 |
| Velocity Threshold | 1.0 | 低于此值忽略碰撞反弹 |
| Max Linear Correction | 0.2 | 默认 |
| Baumgarte | 0.2 | 默认 |
| Simulation Mode | FixedUpdate | 与帧率解耦，物理稳定 |
| Auto Sync Transforms | false | 性能优化 |
| Queries Hit Triggers | false | 普通查询不命中触发器 |

### 1.2 PhysicsMaterial2D（"毡毯材质"）

```
名称：FeltMat
Friction: 0.5      // 毡毯摩擦较大
Bounciness: 0.15   // 羊拐弹性较低，模拟骨头
```

### 1.3 羊拐物理体（ShagaiBody）

| 组件 | 配置 |
|---|---|
| `Rigidbody2D` | BodyType = Dynamic / Mass = 0.5 / LinearDrag = 0.3 / AngularDrag = 0.5 / GravityScale = 1 / CollisionDetection = Continuous（防穿透）|
| `PolygonCollider2D` | 7-9 个顶点的简化轮廓 / Material = FeltMat |
| `ShagaiOrientation` 脚本 | 自定义组件，管理"加权随机最终面" + 形态判定 |

### 1.4 边界
- 场地四周设置 `EdgeCollider2D` 防止飞出
- 顶部不封闭（允许抛得高）
- 边界材质 Bounciness 设为 0（吸收弹力，避免反弹失控）

## 2. 羊拐 2D 模型

由于是 2D，我们用"伪 3D"表现：

### 2.1 形态表现方式
4 个面用 4 张不同贴图，根据当前 `currentFace` 切换 SpriteRenderer：

| Face 枚举 | 贴图 | 含义 |
|---|---|---|
| Sheep (S) | `shagai_face_sheep.png` | 宽凸面 - 绵羊 |
| Goat (G) | `shagai_face_goat.png` | 宽凹面 - 山羊 |
| Horse (H) | `shagai_face_horse.png` | 窄凸面 - 骏马 |
| Camel (C) | `shagai_face_camel.png` | 窄凹面 - 骆驼 |

### 2.2 翻滚动画方式
物理移动时，根据角速度 + 线速度，**循环切换 4 张贴图**模拟翻面（每 0.1~0.2s 切一张）。
- 高速翻滚 → 切换快
- 慢速 → 切换慢
- 静止前 → 锁定为"最终面"

### 2.3 不规则形状
PolygonCollider2D 用一个略不规则的 8 边形（参考真实羊拐俯视轮廓）。
不规则形状让翻滚时停留位置有自然差异（增加视觉真实感），最终面仍由加权随机决定。

## 3. 加权随机决定最终面

### 3.1 概率配置（可配置，外部 JSON）

```json
{
  "DefaultProbability": {
    "Sheep":  0.35,
    "Goat":   0.30,
    "Horse":  0.18,
    "Camel":  0.12,
    "Edge":   0.05
  }
}
```

> `Edge`（侧立）触发后需要"重抛动画"：自动小幅弹跳一次重新判定，避免出现侧立结果（玩家体验差）。

### 3.2 算法

```csharp
public Face RollFinalFace() {
    float r = Random.value;  // [0, 1)
    float acc = 0f;
    foreach (var entry in probabilityTable) {
        acc += entry.Probability;
        if (r < acc) return entry.Face;
    }
    return Face.Sheep;  // 兜底
}
```

### 3.3 调用时机
- 玩家释放抛掷时 → **立即**调用一次 `RollFinalFace()` 决定最终面（结果先存起来）
- 物理模拟过程中正常翻滚
- 检测"近乎静止" → 进入过渡动画 → 收敛到最终面

### 3.4 防作弊（PvP 模式）
- 客户端的加权随机种子由**云函数**派发（玩家 A 与玩家 B 看到相同的种子序列）
- 异步 PvP 时，录像内含种子，回放时复现完全相同结果
- 单机模式可用本地 `Random`

## 4. "几乎静止"检测

```csharp
bool IsAlmostStopped(Rigidbody2D rb) {
    return rb.velocity.sqrMagnitude < 0.01f
        && Mathf.Abs(rb.angularVelocity) < 5f;
}
```

需连续满足 **0.5 秒** 才确认（避免短暂卡顿误判）。

```csharp
private float stableTime = 0f;
void FixedUpdate() {
    if (IsAlmostStopped(rb)) {
        stableTime += Time.fixedDeltaTime;
        if (stableTime >= 0.5f && !isConverging) {
            StartFaceConvergence();
        }
    } else {
        stableTime = 0f;
    }
}
```

## 5. 翻面过渡动画

### 5.1 过渡流程
```
1. 暂停 / 减弱物理（Rigidbody2D.simulated = false 或 Mass × 10）
2. 当前贴图 → 模糊 (0.1s 缩放 X 至 0)
3. 切换为最终面贴图
4. 弹性放大恢复 (0.15s 缩放 X 0 → 1, OutBack)
5. 触发回调 OnSettled(finalFace)
```

### 5.2 实现（DOTween 示例）

```csharp
public void ConvergeToFace(Face target) {
    isConverging = true;
    rb.simulated = false;
    
    spriteRenderer.transform
        .DOScaleX(0f, 0.1f)
        .OnComplete(() => {
            spriteRenderer.sprite = GetSpriteForFace(target);
            spriteRenderer.transform
                .DOScaleX(1f, 0.15f)
                .SetEase(Ease.OutBack)
                .OnComplete(() => OnSettled?.Invoke(target));
        });
}
```

### 5.3 视觉掩盖原则
- 过渡时长 ≤ 0.25 秒，玩家来不及反应"伪物理"
- 加入轻微"灰尘粒子"特效增加自然感
- 多枚羊拐同时抛掷时，错开各自的收敛时刻（±100ms 随机偏移）避免整齐切换暴露

## 7. 多枚羊拐同时模拟

抛沙嘎一次抛 4 枚；弹沙嘎场上多枚。

### 7.1 性能预算
- 单场最多 20 枚物理体（弹沙嘎极限场景）
- 60 FPS 下，FixedUpdate 频率 50Hz → 单帧物理预算 20ms
- 实测目标：单帧物理耗时 ≤ 5ms

### 7.2 优化策略
- **休眠**：`Rigidbody2D.sleepMode = StartAwake / NeverSleep` 慎用，让 Box2D 自动休眠
- **碰撞层**：羊拐 vs 羊拐 / 羊拐 vs 边界 / 羊拐 vs 装饰（仅前两层启用）
- **关闭非必要组件**：静止后 disable `ShagaiOrientation` Update

## 8. 单元测试与统计验证

### 8.1 自动测试场景
建立专用测试场景 `Test_ShagaiPhysics`：
- UI 按钮"批量抛掷 1000 次"
- 每次自动抛掷 4 枚（随机力度方向）
- 统计每次每枚的 `Face` 结果
- 输出统计面板：
  - 各面出现次数与占比
  - 与目标概率的偏差
  - 平均稳定时间
  - 平均触发"侧立重抛"次数

### 8.2 验收指标

| 指标 | 目标 |
|---|---|
| 各面分布与目标概率偏差 | ≤ ±2 个百分点 |
| 平均稳定时间 | ≤ 2 秒 |
| 侧立重抛比例 | ≤ 5% |
| 单帧最大物理耗时 | ≤ 5ms |
| 帧率（抖音真机）| ≥ 50 FPS |

## 10. 失败回退方案

| 风险 | 触发条件 | 退路 |
|---|---|---|
| Box2D 在 Unity WebGL 抖音环境性能差 | 帧率 < 30 FPS | 使用更简化的 BoxCollider2D；减少同时物理体数量 |
| 加权随机被玩家察觉 | 用户反馈"假" | 加大物理"自然停止"权重，仅"侧立"才强制加权 |
| 翻面过渡动画穿帮 | 切换肉眼可见 | 改为粒子遮挡 + 提速到 0.15s |
| 抖音机型性能差异大 | 低端机翻滚不流畅 | 提供"低画质模式"降低物理体数量与精度 |
