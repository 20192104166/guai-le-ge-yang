## 0. 架构总览

```
[Unity 客户端]
     │ tt.* + 字节官方 Unity 转换插件
     ▼
[抖音小游戏运行环境]
     │ HTTPS / WebSocket（仅未来实时 PvP）
     ▼
┌────────────────────────────────┐
│      抖音云开发（DouyinCloud）  │
│ ┌────────────────────────────┐ │
│ │  云函数 (Node.js Faas)     │ │ ← 业务逻辑入口
│ ├────────────────────────────┤ │
│ │  云数据库 (MongoDB-Like)   │ │ ← 玩家 / 排行榜 / 录像
│ ├────────────────────────────┤ │
│ │  云存储 (CDN / 对象存储)   │ │ ← 录像文件、图鉴资源
│ ├────────────────────────────┤ │
│ │  抖音账号系统              │ │ ← 一键登录 / 好友关系
│ └────────────────────────────┘ │
└────────────────────────────────┘
```

## 1. 云数据库 Schema

数据库类型：抖音云数据库（MongoDB 兼容）。每个 collection 对应一个集合。

### 1.1 `users` - 玩家信息

```js
{
  _id: ObjectId,
  openid: String,           // 抖音 openid（唯一）
  nickname: String,         // 抖音昵称
  avatar: String,           // 头像 URL
  createdAt: Number,        // 注册时间戳
  lastLoginAt: Number,
  
  // 资产
  shagaiCount: Number,      // 持有羊拐数（弹/抛通货）
  masterScore: Number,      // 大师分（综合积分）
  
  // 弹沙嘎 ELO
  flickElo: Number,         // ELO 分数，初始 1000
  flickTier: Number,        // 段位 1-6
  flickSeasonId: String,    // 当前赛季
  
  // 抛沙嘎
  tossBestScore: Number,    // 历史单局最高分
  tossWeekBest: Number,     // 当周最高分
  tossWeekUpdatedAt: Number,// 周首次更新时间（用于跨周清零）
  
  // 抓沙嘎
  catchMaxLevel: Number,    // 永久最高通关数
  catchWeekBestTime: Number,// 当周最快通关 5 关用时 (ms)
  catchWeekUpdatedAt: Number,
  
  // 设置
  settings: { sound: Boolean, music: Boolean, vibration: Boolean }
}
```

**索引**：`openid` (unique), `masterScore desc`, `flickElo desc`, `tossWeekBest desc`, `catchMaxLevel desc`

### 1.2 `replays` - PvP 录像

```js
{
  _id: ObjectId,
  ownerOpenid: String,      // 录像主玩家
  gameMode: String,         // "flick" | "toss" | "catch"
  createdAt: Number,
  expiresAt: Number,        // 30 天过期（节省存储）
  
  // 录像内容
  seed: Number,             // 随机种子（确定加权随机序列）
  initialState: Object,     // 初始场景状态（羊拐数量/位置）
  operations: [             // 操作序列
    {
      t: Number,            // 时间戳（相对开局 ms）
      type: String,         // "drag_start" | "drag_release" | "tap" 等
      data: Object          // 操作数据（坐标、力度、方向）
    }
  ],
  result: {
    score: Number,          // 玩家得分
    won: Boolean,           // 是否获胜（PvP 比较用）
    keyEvents: [String]     // 关键事件标签（"撂四样"等）
  },
  
  // PvP 配对
  challengeId: String,      // 挑战 ID（玩家 B 通过此 ID 找到对手录像）
  opponentOpenid: String,   // 对手 openid（被挑战者）
  opponentReplayId: ObjectId,// 对手录像 ID（双方完成后填充）
  pvpStatus: String         // "waiting" | "matched" | "settled"
}
```

**索引**：`ownerOpenid`, `challengeId`, `expiresAt` (TTL)

### 1.3 `leaderboards` - 排行榜快照

> 实时榜单可由 users 表索引查询；本表存"赛季归档"快照。

```js
{
  _id: ObjectId,
  type: String,             // "flick_elo" | "toss_week" | "catch_week" | "master"
  seasonId: String,         // 赛季 ID（如 "2026-06"）或 weekId
  rank: Number,             // 名次
  openid: String,
  nickname: String,
  avatar: String,
  score: Number,            // 该榜单的分数值
  tier: Number,             // 段位（仅 flick_elo 有）
  snapshotAt: Number        // 快照时间
}
```

**索引**：`type + seasonId + rank`

### 1.4 `skins` - 图鉴：羊拐皮肤

```js
{
  _id: ObjectId,
  skinId: String,           // 唯一 ID，如 "skin_inner_mongolia_classic"
  region: String,           // "内蒙古" | "外蒙古" | "青海" | "新疆" | "西藏" | "达斡尔" | ...
  name: String,             // 显示名
  description: String,      // 文化背景描述
  rarity: Number,           // 1-5 星
  texture4Faces: {          // 4 面贴图 URL
    sheep: String, goat: String, horse: String, camel: String
  },
  unlockCondition: Object,  // { type: "auto"|"achievement"|"ad", value: ... }
  createdAt: Number
}
```

### 1.5 `cultureCards` - 图鉴：文化卡片

```js
{
  _id: ObjectId,
  cardId: String,           // 唯一 ID，如 "card_morin_khuur"
  category: String,         // "animal" | "instrument" | "pattern" | "festival" | "legend"
  name: String,
  description: String,      // 长文化描述
  imageUrl: String,
  rarity: Number,           // 1-5
  unlockCondition: Object,
  createdAt: Number
}
```

### 1.6 `userInventory` - 玩家拥有的皮肤/卡片

```js
{
  _id: ObjectId,
  openid: String,
  itemType: String,         // "skin" | "card"
  itemId: String,           // 对应 skinId / cardId
  acquiredAt: Number,
  currentlyEquipped: Boolean// 仅 skin 用
}
```

**索引**：`openid + itemType`, `openid + itemId` (unique)

### 1.7 `friendships` - 好友关系（缓存）

```js
{
  _id: ObjectId,
  openid: String,
  friendOpenid: String,
  syncedAt: Number          // 上次从抖音 SDK 同步时间
}
```

**索引**：`openid`

## 3. PvP 异步录像协议

### 3.1 录像格式（详见 1.2 `replays` Schema）

核心三段：
- **seed**：所有"加权随机"使用此种子，保证双方回放完全一致
- **initialState**：开局参数（如抛沙嘎场地配置、双方羊拐数）
- **operations[]**：玩家操作序列（带时间戳）

### 3.2 异步 PvP 流程

```
玩家 A 开局
  ├ 客户端调云函数 `pvp_create_challenge`，传 gameMode + targetOpenid（可空表示开放挑战）
  ├ 服务端生成 seed，存录像
  └ 返回 challengeId
玩家 A 进行游戏
  └ 客户端按 seed 模拟、记录所有 operations
  ↓
玩家 A 上传录像
  └ `pvp_create_challenge` 已完成时调用上传补丁，保存 result

玩家 B 收到通知（推送 / 列表 `pvp_list_pending`）
  ├ 客户端调 `pvp_accept_challenge` → 拉取对手录像
  ├ 本地"幽灵回放"对手录像（半透明显示对手操作过程）
  └ 玩家 B 同步使用相同 seed 进行操作
玩家 B 完成
  └ `pvp_submit_response` 上传 B 的录像 + 结果
服务端裁定胜负
  ├ 比较 A、B 的 result.score
  ├ 更新双方 ELO / 大师分
  └ 推送结算给双方
```

### 3.3 防作弊
- 客户端只能上传录像，不能直接上传"我赢了"
- 服务端云函数用 A 的 seed + operations **重新模拟一次** 验证 result
- 同一玩家短时间多次创建挑战 → 限流
- 录像数据加签（防中间人篡改）

## 4. 客户端 SDK 接入

### 4.1 Unity 端封装层
建议在 Unity 中封装一层 `CloudService` 静态类：

```csharp
public static class CloudService {
    public static async Task<User> LoginAsync();
    public static async Task<User> GetMyProfileAsync();
    public static async Task<MatchResult> SubmitFlickMatchAsync(Replay replay, ...);
    public static async Task<List<LeaderboardEntry>> QueryLeaderboardAsync(LeaderType type, int from, int to);
    // ...
}
```

底层通过抖音云开发提供的 JavaScript Bridge 调用云函数。Unity 通过 `Application.ExternalCall` 或 jslib 桥接。

### 4.2 错误处理
- 网络错误 → 本地重试 3 次，仍失败提示"网络不佳，请稍后再试"
- 服务端错误 → 上报 metrics + 友好提示
- 鉴权失败 → 自动重新登录

### 4.3 离线缓存
- 个人资产、上次榜单结果本地缓存
- 玩家可离线进入单机模式
- 联网时增量同步

## 5. 部署与运维

### 5.1 环境
- **dev**：开发环境，单独数据库
- **prod**：生产环境

### 5.2 监控
- 抖音云开发自带监控（云函数调用次数、错误率、延迟）
- 自定义 metrics 上报（崩溃、性能、关键业务指标）

### 5.3 备份
- 用户数据每日自动备份
- 重要 collection（users / userInventory）定期导出

### 5.4 成本控制
- 录像 TTL 30 天（自动清理）
- 头像、皮肤贴图走 CDN + 长缓存
- 单玩家月活成本 < ¥0.1 目标

---

## 6. 风险与缓解

| 风险 | 缓解 |
|---|---|
| 抖音云开发额度超限 | 监控用量；接入告警；准备迁移自建预案 |
| 录像存储成本失控 | 严格 TTL + 仅高分录像归档 |
| 高并发周榜重置 | cron 用分桶 + 流量控制 |
| 云函数冷启动延迟 | 关键函数定期保温调用 |
